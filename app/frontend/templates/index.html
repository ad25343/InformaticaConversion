The 12-Step Pipeline
Step 1Parse Report
Reads your XML. Extracts all mappings, sources, targets, and transformations.
Step 2Classify
Rates complexity: Low / Medium / High / VeryHigh based on transformation count.
Step 3Document
Auto-generates source-to-target lineage docs and a full S2T mapping matrix.
Step 4Verify
Runs completeness, accuracy, and self-checks. Flags blockers and recommendations.
Step 5 ⏸Analysis Sign-off
Business analyst reviews docs and signs off before code generation begins.
Step 6Stack Assign
Picks the best target stack: Python, PySpark, or dbt based on complexity.
Step 7Convert
Generates production-ready code using AI with any manifest overrides applied.
Step 8Security Scan
Scans generated code for vulnerabilities using Bandit + pattern matching.
Step 9 ⏸Security Sign-off
Security architect reviews scan findings and approves or requests fixes.
Step 10Quality Review
Checks logic equivalence, coverage, and code quality against standards.
Step 11Test Generation
Generates unit tests with field-level coverage reporting.
Step 12 ⏸Final Sign-off
Final human approval. Once approved, job is Complete and artifacts are exported.
💡 Individual mode tips:
• Use Individual for a single mapping XML export from Informatica Designer.
• Optionally attach the matching Workflow XML to extract session-level metadata.
• Parameter files (.txt/.par) provide runtime variable values used in the mapping.
